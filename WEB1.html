태그

웹을 지배하는 문법을 태그라고 합니다.

HTML은 "HyperText Markup Language"의 약자 입니다.
html을 만든 사람들은 제목을 헤드"head"태그로, 본문을 바디"body"태그로 묶기로 약속 하였습니다.
따라서, html에있는 모든 태그는 반드시 헤드 또는 바디 태그 중 하나 아래에 놓입니다.
그런점에서 헤드태그와 바디태그는 고위직 태그라고 할 수 있다.
그러한 두 고위직 태그들을 감싸는 단 하나의 최고위층 태그가 "html"태그 입니다.
그리고 "html"태그 위에 관용적으로 "이 문서는 html이다"라는 뜻에서 "!doctype html"라고 씁니다.

utf-8fh이라는 방식으로 저장이 되어있다면 같은 방식으로 열어야만 문자에 오류가 생기지 않습니다.
[meta charset="utf-8fh"] = utf-8fh로 문서를 읽어라.
"character"는 문자 라는 뜻이고 "set"은 규칙 이라는 뜻입니다.

웹페이지의 이름을 정할때는 타이틀"title"태그를 사용합니다.
타이틀은 책의 표지와 같은 역할을 합니다.

제목을 정할때는 에이치"h"태그 를 사용합니다.
제목"head"의 약자이며 h1 ~ h6 까지있고 숫자가 클수록 글씨는 작아집니다.

문자를 강조(두껍게) 표현할때는 스트롱"strong" 태그를 사용합니다.

문자밑에 밑줄을 넣고 싶을때는 유"u"태그를 사용합니다.

줄을 바꾸고 싶다면
비알"br"태그를 사용합니다.

단락을 나누어 표현 하고싶다면 피"p"태그를 사용합니다.

이미지를 넣고싶다면 이미지"img"태그를 사용합니다.
"width"소스를 100%로 설정하면 윈도우 크기에 맞춰 변합니다.

img태그 뒤에 어떤 이미지 인지 알려주는 속성은 소스"source"입니다. 줄여서"src"라고도 씁니다.
[scr=""],[width""] 이런것들을 속성"attribute"이라고 부릅니다. 속성은 위치에 상관이 없습니다.
태그에 태그만으로 정보가 부족할때 이러한 속성으로 더 많은 의미를 부여할 수 있습니다.

  • 목록을 나타내고 싶다면 리스트"list"태그를 사용합니다.
  • 자식인 "li"태그는 반드시 부모인 "ul"태그를 가지고 있습니다.
  • 예시 : [parent] [child] [/child] [/parent] = [ul] [li] [/li] [/ul]
  • "ul"은 목록"list"에서 점으로 나타내지만 "ol"을 사용한다면 숫자로 나타냅니다.
  • "ul"은 "unorbered list"의 약자 이고, "ol"은 "Ordered List"의 약자 입니다.

텍스트에 링크를 걸고 싶을때는 앵커"anchor"라는 뜻의 "a"태그를 사용합니다.
"a"태그 뒤에 앵커를 내릴 주소 링크참조 속성"href"를 부여합니다. ("hypertext" = 링크) + ("reference" = 참조) = "hret"
링크를 새 탭으로 열고 싶다면 [target="_blank"] 속성을, 링크에게 이름을 부여하고 싶다면 [title=""] 속성을 사용합니다.
예시 : [a href="링크" target="_blank" title="이름"]



인터넷

인터넷이 완성 되기 위해서는 컴퓨터 두대가 필요합니다.
웹브라우저가 깔려있는 컴퓨터와, 웹서버가 깔려있는 컴퓨터.
웹브라우저는 웹서버에 정보를 요청합니다. 웹서버는 웹브라우저의 요청에 응답합니다.
웹브라우저는 고객"client"이고, 웹서버는 서비스를 제공하는 사람"server" 라고 합니다.

인터넷을 하는데에는 두가지 방법이 있습니다.

  1. 자신의 컴퓨터에 웹서버를 직접 깔아서 하는 것. "web server"
    자신의 컴퓨터에 직접 설치하는 것은 어렵습니다.
    하지만 그 과정에서 인터넷과 관련된 여러 가지 개념을 배울 수 있습니다.

  2. 이런 일을 대행해 주는 대행업체한테 맡기는 것. "web hosting"
    웹 호스팅이라는 것을 이용해서 회사에 맡기는 것은 쉽습니다.
    하지만 인터넷 이라는 것이 동작하는 원리와 같은 것들이 많이 감춰져 있기 때문에 배울 것이 적습니다.

웹 호스팅

웹서버를 운영하는 일은 쉬운일이 아닙니다.
우선, 컴퓨터가 있어야 하고 컴퓨터가 냉장고처럼 항상 켜져있어야 하고 인터넷이 끊기지도 않아야 합니다.
웹서버 라는 프로그램을 배워서 설치하고 잘 운영할 수 있어야 합니다.
인터넷을 통해서 여려분의 정보를 웹으로 전송할 수 있게 설정도 해야합니다.
이런 일들은 하나하나가 다 쉬운일이 아닙니다.
그래서 이러한 일들을 대행해주는 회사들이 있습니다.
인터넷에 연결된 컴퓨터 하나하나를 호스트"host"라 하고, 이런 컴퓨터를 빌려주는 사업을 호스팅"hosting"이라 합니다.
또는 요즘 클라우드"cloud"라고도 부릅니다. 인터넷을 빌려주는 비지니스의 점에서는 같은 개념입니다.
호스팅 중에서, 컴퓨터의 웹서버와 같이 웹을 동작하기 위해서 필요한 소프트웨어들 까지 설치해서 빌려주는 비지니스를 웹호스팅"web hosting"이라고 합니다.


GitHub

내 소스코드 인터넷에 저장하기
깃허브"GitHub"는 원래 웹포스팅을 위해서 고안된 사이트는 아닙니다.
이 사이트는 개발자들이 자신의 소스코드를 안전하게 백업하고 백업된 소스코드를 매개로 개발자들끼리 협업을 하기위해서 고안된 사이트입니다.
이 사이트의 기능중에 웹포스팅 기능이 있기때문에 이 사이트를 활용 할 것입니다.
가장 먼저 할것을 우리의 소스코드를 저장할 공간인 저장소 파지토리"pository"를 만들어야 합니다.
파지토리를 만든후, 우리는 우리가 만든 소스코드를 깃허브닷컴에 업로드 합니다.
"uploading an existing file"링크를 클릭해 내 파일을 넣습니다.
그 후 밑에 오늘 한 작업의 이름을 넣고 "Commit changes"를 클릭 하면 여러분이 작업한것에 대한 첫번째 버전이 인터넷에 안전하게 저장이 됩니다.
저장된 파일을 클릭해 내용을 확인할 수 있고 그 페이지의 주소를 공유하면, 자신의 소스코드 자체를 다른사람과 공유하고 토론도 할 수 있습니다.

저장한 내 소스코드로 웹사이드 운영하기
깃허브닷컴에서는 우리가 보관하고 있는 소스코드를 이용해서 웹사이트를 운영할 수 있는 기능을 제공 합니다.
깃허브 안에 내장되어 있는 깃허브 페이지라는 기능입니다.
세팅"setting" > 페이지스"pages" > 소스"source" > none - main or master > 세이브"save"
엑션스"Actions"메뉴로 들어가면 저장소에서 일어나고 있는 여러가지 작업들이 어떻게 진행되고 있는지 확인할 수 있습니다.
빌드"build"과정이 끝나면 디플로이"deploy"과정이 시작 됩니다. 디플로이는 배치하는 과정 입니다.
만약 파일의 내용을 바꿨다면 저장소의 홈으로 가서 오른쪽의 애드파일"Add File" > 업로드파일"Upload File"을 클릭하여 같은 작업을 반복합니다.

웹 호스팅

"http"는 "Hyper Text Transfer Protocol"의 약자입니다. 웹 브라우저는 제품명이 아니라 제품군 입니다. 웹서버도 마찬가지 입니다. 웹서버 라는 제품군이 있고 여기에는 여러가지 제품들이 있습니다. "apache", "iis", "nginx"등등등 웹브라우저 만큼 다양한 웹 서버가 있습니다. 웹서버중의 하나인 라이브서버"Live Server"는 "visual studio code"의 확장 기능입니다. 하단의 "go live"를 클릭하여 "Port:5500"이 뜨면 웹서버가 실행된 것입니다. 동시에 웹브라우저가 실행될 것이고 주소는 "http://127.0.0.1:5500/"입니다. 라이브서버의의 진짜 유용함은 실시간 편집 기능입니다. 코드를 편집하면 실시간으로 내용이 변경 됩니다. 코드를 수정하고 저장하면 리로드를 할 필요가 없어집니다. 웹서버를 끄기 위해서는 포트"port"를 클릭하면 됩니다.

"127.0.0.1"은 인터넷프로토콜"internet protocol"에서 사용하는 주소"ip"입니다. "ip adress"는 약 43억개가 존재 하는데, 그 중 가장 특별한 주소가 "127.0.0.1"입니다. 이 주소는 내 컴퓨터 자신을 가르키자고 전세계적으로 약속한 주소 입니다. 그래서 지금 우리의 웹브라우저는 같은 내 컴퓨터의 웹서버에 접속을 시도하고 있는 것입니다. 하나의 컴퓨터에 여러개의 서버가 동작하고 있다면, 웹브라우저는 누구와 통신해야 할까요. 그래서 각각의 서버는 각자의 포트번호를 가지고 있습니다. (웹서버 5500 / 게임서버 6000 / 채팅서버 7000) 웹브라우저에 우리가 입력한 주소"ip"의 포트번호가 5500번 일때, 웹브라우저는 5500번 포트에 연결 되어있는 웹서버와 통신할 수 있게 됩니다. 서버가 있는 컴퓨터의 주소"ip"를 포트번호와 함께, 같은 인터넷에 연결 되어있는 컴퓨터의 웹브라우저에 입력하고 접속을 시도하면 웹페이지가 표시 됩니다.
